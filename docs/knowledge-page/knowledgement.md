## what is Knowledgement
当用户输入一个知识点，页面就可以请求 llm 生成知识点大纲，生成知识点大纲后，再并行为每个大纲生成题目。用户可以通过页面看整体的全流程，包括耗时，生成的大纲和题目内容。
整体是流失的，所以页面也要用流式的方式展示

## 为什么要有这个页面
这个页面主要方便管理员测试产品来用，背景是这样手机软件不便于测试，而这个页面可以让用户非常直观的看到整体的流程以及消耗的 token

## Userflow
用户输入知识点 -> 弹出弹窗 -> 弹窗上面展示当前在哪一个步骤，然后下面流式展示大纲生成过程 -> 大纲生成完毕后，在并行展示每个大纲生成的状态。当全部生成完成，会提示用户，用户就可以看到消耗的耗时等元信息(我一会会告诉你元信息有哪些)和生成题目内容了

## 元信息
id: 题目 id
title: 用户输入的知识点
create_time: 创建时间
model: 用户选择的模型
status: 当前题目的状态（开始|大纲生成中|大纲生成完毕| 题目生成中 | 成功 | 失败）
error_msg: 如果失败了，一定要给原因
time_consume: 一共消耗了多少时间
input_token: 输入的 token 数量
output_token: 输出的 token 数量
